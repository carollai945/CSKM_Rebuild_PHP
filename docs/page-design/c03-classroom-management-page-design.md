# C03 教室管理 頁面設計

## 1. 基本資料

- 頁面：`C03ClassroomView.vue`（路由：`/classroom-management`）
- 對應 BDD / SDD / TDD：`c03-classroom-management`
- 使用者：教務人員、管理員

## 2. 頁面目的

- 依區域查詢教室清單
- 新增、修改與刪除教室資料（名稱、容量、狀態）

## 3. 版面區塊

| 區塊 | 說明 |
|---|---|
| 篩選列 | 區域下拉選單與查詢按鈕 |
| 教室清單 | 顯示教室名稱、容量、狀態，提供編修與刪除按鈕 |
| 新增教室區 | 區域、名稱、容量、狀態輸入欄位與新增教室按鈕 |
| 訊息區 | 成功訊息（綠色）與錯誤訊息（紅色） |

## 4. 欄位與顯示規則

| 欄位 | 輸入類型 | 必填 | 說明 |
|---|---|---|---|
| 區域（篩選） | 下拉選單 | 否 | 選「全部區域」時顯示所有教室 |
| 區域（新增） | 下拉選單 | 是 | 選擇教室所屬區域 |
| 教室名稱 | 文字輸入 | 是 | 輸入教室名稱 |
| 容量 | 數字輸入 | 是 | 最小值 1 |
| 狀態 | 下拉選單 | 是 | `ACTIVE`（啟用）/ `INACTIVE`（停用），預設啟用 |

## 5. 按鈕與動作

| 按鈕 | 位置 | 動作 |
|---|---|---|
| 查詢 | 篩選列 | 清空清單，依選取區域重新載入 |
| 編修 | 教室清單列 | 切換該列為可編輯狀態 |
| 儲存 | 教室清單列（編輯中） | 送出 PUT 更新並刷新清單 |
| 取消 | 教室清單列（編輯中） | 取消編輯，還原顯示狀態 |
| 刪除 | 教室清單列 | 送出 DELETE 並刷新清單 |
| 新增教室 | 新增教室區 | 驗證欄位後送出 POST 並刷新清單 |

## 6. 互動規則

- 進入頁面時自動載入區域清單與全部教室清單
- 切換區域下拉後立即重新查詢（`@change` 觸發）
- 按查詢按鈕時先清空清單再重載
- 新增成功後清空新增表單欄位
- 同一時間只允許一列處於編輯狀態

## 7. 訊息設計

| 情境 | 訊息 |
|---|---|
| 新增成功 | 教室新增成功 |
| 修改成功 | 教室修改成功 |
| 刪除成功 | 教室刪除成功 |
| 未選區域 | 請選擇區域 |
| 未填名稱 | 請輸入教室名稱 |
| 容量不足 | 容量至少為 1 |
| 載入失敗 | 載入教室清單失敗 |
| 新增失敗 | 新增教室失敗 |
| 修改失敗 | 修改教室失敗 |
| 刪除失敗 | 刪除教室失敗 |

## 8. 權限與狀態

- 所有登入使用者可查詢（GET）
- 僅具 `ROLE_ACADEMIC` 或 `ROLE_ADMIN` 角色者可新增、修改、刪除
- 無權限操作時後端回傳 403，前端顯示錯誤訊息
