# C02 學生管理（主檔）頁面設計

## 1. 基本資料

- 頁面：`C02StudentManagementView.vue`（路由 `/student-management`）
- 對應 BDD / SDD / TDD：`c02-student-management`
- 使用者：教務人員、業務人員、管理員

## 2. 頁面目的

- 查詢學生主檔資料
- 新增學生主檔
- 匯出學生清單為 Excel

## 3. 版面區塊

- **查詢列**：姓名輸入框、區域下拉、機構下拉、課程下拉、查詢按鈕、匯出按鈕
- **操作列**：新增學生按鈕、編修按鈕（需先勾選學生）
- **學生列表**：含全選 checkbox、學號、姓名（可點擊進入明細）、區域、機構、課程、學顧、手機、Email、等級
- **新增學生彈窗**：姓名、手機、性別、區域、機構、電話、Email、等級（StuLevel）、來源（StuSource）、建檔日期

## 4. 欄位與顯示規則

- 姓名欄位可點擊連結，導向 C046 學員明細頁（`/students/:studentId`）
- 區域、機構、課程下拉選項由後端依登入者權限動態提供
- CEO 可查看所有區域；主管僅能查看本區域；業務人員僅能查看授權範圍內的學生
- 匯出按鈕帶入目前查詢條件，呼叫 `GET /api/v1/students/export`

## 5. 按鈕與動作

- **查詢**：清空列表，依目前條件重新載入
- **匯出 Excel**：下載符合目前篩選條件的 `.xlsx` 檔案
- **新增學生**：開啟彈窗表單
- **編修**：必須先勾選至少一筆學生；帶入第一筆勾選學生 id 進入 C046 明細頁
- **彈窗儲存**：前端驗證必填欄位後送出 `POST /api/v1/students`
- **彈窗取消**：關閉彈窗

## 6. 互動規則

- 進入頁面時自動執行一次查詢
- 按下查詢前先清空前一次搜尋結果與勾選狀態
- 未勾選學生時編修按鈕為 disabled 狀態
- 新增成功後自動關閉彈窗並重新執行查詢

## 7. 訊息設計

- 姓名為必填欄位
- 手機為必填欄位
- 區域為必填欄位
- 日期格式錯誤，請使用 YYYY-MM-DD
- 儲存成功 / 查詢失敗 / 新增失敗

## 8. 權限與資料範圍

- CEO（ADMIN）：可查看並操作所有區域的學生
- 主管（SUPERVISOR）：僅能查看本區域學生
- 業務人員（STAFF）：僅能查看指定給自己或有授權的學生
