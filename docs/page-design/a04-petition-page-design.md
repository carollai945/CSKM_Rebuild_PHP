# A04 簽呈申請 / D01 審核流程 頁面設計

## 1. 基本資料

- 頁面：`A04.jsp`、`A040.jsp`、`D01.jsp`、`D010.jsp`
- 對應 BDD / SDD / TDD：`a04-petition`
- 使用者：申請人、區域主管、CEO

## 2. 頁面目的

- 建立簽呈並完成主管與 CEO 的分層審核

## 3. 版面區塊

- 申請列表區：查詢條件、歷史清單、建立入口
- 簽呈明細區：主旨、內容、附件、狀態
- 審核列表區：待審清單、批次回覆區
- 審核明細區：內容檢視、意見欄位、核准/退回按鈕

## 4. 欄位與顯示規則

- 列表顯示快捷單號、主旨、狀態與入口
- 審核頁依角色切換主管或 CEO 動作
- 已送審資料對申請人鎖定編修

## 5. 按鈕與動作

- `儲存草稿`、`送審`
- `核准`、`退回`
- `批次回覆`、`全選`、`全不選`

## 6. 互動規則

- 主旨關鍵字查詢套用三個月限制
- 批次回覆依角色帶不同審核類型
- 失敗時列出未成功處理的單號

## 7. 訊息設計

- 必填欄位未完成
- 退回需填意見
- 批次處理成功/失敗摘要

## 8. 權限與狀態

- 申請人只處理自己的簽呈
- 主管只審核主管階段，CEO 只審核最終階段
