# D03 報表審核 頁面設計

## 1. 基本資料

- 頁面：`D03.jsp`
- 關聯明細頁：`D030.jsp` ~ `D033.jsp`
- 對應 BDD / SDD / TDD：`d03-report-approval`
- 使用者：主管、CEO

## 2. 頁面目的

- 依報表類型進入對應明細頁，完成工作報表審核

## 3. 版面區塊

- 查詢條件區：日期區間、填報人、報表類型單選（日報 / 週報）、查詢按鈕
- 學顧報表表格區：`D03_Spu_table`，結果列由 Ajax 依報表類型動態 append
- 行政報表表格區：`D03_Adm_table`，結果列由 Ajax 依報表類型動態 append
- 關聯明細審核頁：依報表型別導向對應 `D030` ~ `D033`

## 4. 欄位與顯示規則

- 兩張表會依查詢類型切換 caption，例如學顧日報 / 行政日報或學顧週報 / 行政週報
- 表格顯示日期或週期、填報人、事由、主管意見、CEO 核示與檢視按鈕
- 查無資料時兩張表各自顯示 `--`
- 部門下拉會依區域變更以 Ajax 重新塞入 `#dept`

## 5. 按鈕與動作

- `查詢 Search`：以日期、姓名、日報 / 週報條件重建兩張表
- `檢視 View`：以報表編號與類型帶入明細頁

## 6. 互動規則

- Ajax 查詢前會先清空兩張表的舊資料
- 同一批結果依 `staff_type` 拆到學顧與行政兩張表
- 日報與週報切換時，同步變更 caption 與日期欄標題
- 每列 `檢視 View` 按鈕由前端依 `report_no`、報表類型與人員類型動態組成

## 7. 訊息設計

- 報表類型異常
- 意見缺漏
- 審核成功

## 8. 權限與狀態

- 主管與 CEO 僅處理其階段待審資料
