# C04 學員管理 頁面設計

## 1. 基本資料

- 頁面：`C04.jsp`
- 關聯頁面：`C041.jsp`、`C042.jsp`、`C044.jsp`、`C046.jsp`
- 對應 BDD / SDD / TDD：`c04-student-management`
- 使用者：教務、學顧、業務

## 2. 頁面目的

- 維護學員基本資料、課程狀態、繳費與學顧分配

## 3. 版面區塊

- 查詢條件區：區域、機構、課程、學生姓名與搜尋按鈕
- 學員結果表格區：`stuInfoTb`，結果列由 Ajax 查詢後動態 append
- 功能入口欄位區：每列提供 `檢視繳費`、`檢視修課`、`檢視意見`，由 JavaScript 依學員資料組出
- 操作按鈕區：`學顧變更`、`修改資料`
- 關聯子頁區：由 `C04` 帶入 `std_no` 進入其他子頁

## 4. 欄位與顯示規則

- 搜尋結果顯示學員姓名、手機、電話、Email 與三個功能欄位
- 區域、機構、課程為連動下拉
- `修改資料` 與 `學顧變更` 需先勾選目標學員
- `institute`、`course`、`std_no` 選單會隨上層條件改變而以 Ajax 重建

## 5. 按鈕與動作

- `查詢 Search`：重新查詢學員
- `檢視繳費`：進入 `C042`
- `檢視修課`：進入 `C044`
- `檢視意見`：進入 `C041`
- `學顧變更`：對勾選學員啟動變更流程
- `修改資料`：進入 `C046`

## 6. 互動規則

- 區域 / 機構 / 課程連動刷新下層選單
- Ajax 搜尋會清空舊資料再重建 `stuInfoTb`
- 未勾選學員就按 `修改資料` 時顯示提示
- 結果列中的各功能 form 與 hidden 欄位由前端依回傳 JSON 動態建立

## 7. 訊息設計

- 未選課程不得建立繳費
- 金額或日期格式錯誤
- 儲存成功

## 8. 權限與狀態

- 不同角色顯示不同子頁與可編修欄位
- 僅可維護授權範圍內學員
