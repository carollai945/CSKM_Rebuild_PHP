# E00 繳費記錄 頁面設計

## 1. 基本資料

- 頁面：`E00.jsp`
- 對應 BDD / SDD / TDD：`e00-payment`
- 使用者：財務、教務

## 2. 頁面目的

- 建立學生繳費記錄並完成財務 / 教務雙重確認

## 3. 版面區塊

- 查詢條件區：姓名、區域、課程、起迄日期、查詢按鈕
- 結果表格區：`E00_table`，每筆繳費列與確認按鈕由 Ajax 搜尋後動態 append
- 退回理由區：`div_reject`，含退回原因與 `退回 Reject`
- 雙重確認欄位區：每列顯示財務核示與教務核示

## 4. 欄位與顯示規則

- 列表顯示日期、學員、課程、金額、學顧、財務核示、教務核示與備註
- 日期條件使用 datepicker
- 財務與教務依權限看到不同的 `確認` 按鈕
- 退回理由輸入區預設隱藏，只有特定流程才展開
- `institution`、`staff_no`、`course` 會依上層條件以前端程式動態重建選項

## 5. 按鈕與動作

- `查詢 Search`：依條件載入繳費資料
- `確認`（財務）：執行財務核示
- `確認`（教務）：執行教務核示
- `退回 Reject`：填寫退回原因後退回資料

## 6. 互動規則

- 查詢後以 Ajax 重建 `E00_table`
- 財務與教務確認分開判斷，兩邊都完成才算完整確認
- 執行確認成功後立即重新查詢
- 前端會依登入角色動態組出可點擊的核示 checkbox / button 欄位

## 7. 訊息設計

- 金額格式錯誤
- 重複確認
- 建立成功 / 確認成功

## 8. 權限與狀態

- 財務與教務依職能分工確認
