# B00 電訪名單管理 頁面設計

## 1. 基本資料

- 頁面：`B00.jsp`
- 對應 BDD / SDD / TDD：`b00-lead-management`
- 使用者：業務人員、區域主管

## 2. 頁面目的

- 以 Vue 頁面查詢、建立、編修、派發與刪除電訪名單

## 3. 版面區塊

- 查詢區：姓名、電話、手機、Email、派發日期、來源、是否已派發、所屬區域與負責人；多個欄位以 Vue `v-model` / `v-select` 呈現
- 結果表格區：`searchResult`，由 Vue `v-for="customerlist"` 動態渲染勾選框、名單摘要與單筆編輯 / 刪除
- 顯示筆數區：表格 caption 中的 `顯示幾筆`
- 批次派發區：全選 / 全不選、派發對象選單、派發按鈕
- 批次新增刪除區：新增名單、批次刪除
- `customerEditModal`：新增 / 編輯名單共用 modal，內含多個 Vue `v-select` 與條件顯示欄位

## 4. 欄位與顯示規則

- 列表顯示姓名、聯絡方式、來源、教育程度、區域、負責人、建立人與派發資訊
- `view_num` 變更後即以 Vue 重新查詢資料
- 教育程度選擇其他時顯示自填欄位
- modal 新增時清空，編輯時帶入既有資料
- `刪除 Delete` 單筆按鈕只在主管或資料建立者的列上顯示

## 5. 按鈕與動作

- `搜尋`：依目前條件查詢
- `新增名單`：開啟新增 modal
- `編輯 Edit`：開啟編輯 modal
- `派發 Assign`：將已勾選名單批次派給指定學顧
- `刪除 Delete`：刪除單筆或批次資料
- `全選 / 全不選`：更新 Vue 勾選集合

## 6. 互動規則

- 查詢欄位以 `v-model` 維持條件狀態
- 調整每頁顯示筆數會立即刷新列表
- 全選 / 全不選直接同步 `pickCustomer`
- 只有主管可用批次派發與批次刪除
- modal `確認 Save` 需通過前端驗證才可點擊
- Vue 會依權限動態切換 `v-select` 可用狀態、批次按鈕可見性與結果列操作按鈕

## 7. 訊息設計

- 必填欄位缺漏
- 未勾選名單不得批次操作
- 派發成功 / 刪除成功 / 權限不足

## 8. 權限與狀態

- 建立者可編修自己的資料
- 僅主管或原建立者可刪除單筆資料
