# F02 人員與權限維護 頁面設計

## 1. 基本資料

- 頁面：`F02.jsp`
- 對應 BDD / SDD / TDD：`f02-permission-management`
- 使用者：系統管理員、區域主管

## 2. 頁面目的

- 新增員工、調整角色與設定 menu 權限

## 3. 版面區塊

- 查詢條件區：員工姓名 autocomplete、區域、部門、人員與查詢按鈕
- 人員資訊區：目前人員名稱、部門職稱異動、到職 / 離職日期，內容會在查詢後以 JavaScript 動態回填
- 六大權限矩陣區：個人專區、學顧業務管理、教務管理、行政管理、財務管理、系統管理
- 角色勾選區：執行長、區域主管、部門主管、財務主管、督課人員、行政人員
- 管理部門區：`showDepByReg`，含全選與可管理部門清單，由 JavaScript 動態組出 checkbox
- 操作區：新增人員、儲存、刪除、重設密碼

## 4. 欄位與顯示規則

- 姓名欄位輸入關鍵字後須從 autocomplete 清單選人
- 各模組權限區都有自己的 `全選(ALL)` 勾選框
- 區域主管與部門主管角色互斥
- 部門主管需至少選一個管理部門
- 人員資料載入後才顯示到職 / 離職日期與部門職稱異動區
- `department`、`title`、`staff`、`staff_rdt` 的選項會依區域 / 部門 / 人員選擇動態重建

## 5. 按鈕與動作

- `查詢 Search`：載入選定人員的現有權限
- `新增人員 New`：建立新帳號
- `儲存 Save`：保存角色與 menu 權限
- `刪除 Delete`：刪除人員
- `重設密碼 Reset`：重設該人員密碼

## 6. 互動規則

- 姓名欄位提供 autocomplete，選取後可回填區域 / 部門 / 人員資訊
- 勾選部門主管時顯示管理部門區塊；取消時同步隱藏
- 每個模組區的 `全選(ALL)` 會同步勾選該群組所有功能
- 儲存前需通過角色與管理部門檢核
- 查詢後會動態更新 `staffNow`、職稱異動清單、到離職日期與管理部門 checkbox 清單

## 7. 訊息設計

- 姓名或必要資料缺漏
- 未選管理部門
- 儲存成功 / 權限不足

## 8. 權限與狀態

- 管理者可維護全部
- 區域主管僅維護授權範圍
