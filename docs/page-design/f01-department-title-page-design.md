# F01 部門與職稱管理 頁面設計

## 1. 基本資料

- 頁面：`F01.jsp`、`F010.jsp` ~ `F012.jsp`
- 對應 BDD / SDD / TDD：`f01-department-title`
- 使用者：系統管理員

## 2. 頁面目的

- 維護部門與職稱主資料，供 F02 人員與權限維護頁動態選單使用

## 3. 版面區塊

- 部門列表區：顯示部門代碼、部門名稱、所屬區域、狀態
- 職稱列表區：顯示職稱代碼、職稱名稱、所屬部門、狀態
- 編修區：新增或修改部門 / 職稱資料
- 操作區：新增、編修、儲存

## 4. 欄位與顯示規則

- 列表欄位：代碼、名稱、狀態，並顯示對應上層資料（區域或部門）
- 部門代碼：必填且不可重複
- 部門名稱：必填
- 職稱代碼：必填且不可重複（同部門下不得重複）
- 職稱名稱：必填
- 狀態：啟用 / 停用
- 編修區會依目前操作對象切換為部門欄位或職稱欄位

## 5. 按鈕與動作

- `新增`：建立新的部門或職稱資料
- `編修`：載入選取資料到編修區
- `儲存`：送出新增或編修資料
- `停用`：將部門或職稱改為停用（需先通過引用檢查）

## 6. 互動規則

- 儲存前檢查必填欄位與代碼唯一性
- 停用前需檢查是否仍被人員資料引用，若被引用則不可停用
- 儲存成功後清單即時刷新並顯示最新狀態
- 非系統管理員不得看到新增、編修、儲存、停用操作

## 7. 訊息設計

- 代碼重複
- 必填欄位未填寫
- 已被引用不得停用
- 儲存成功
- 權限不足

## 8. 權限與狀態

- 僅系統管理員可維護
- 狀態需遵循啟用 / 停用兩種顯示與儲存值
