# A02 個人報表入口 頁面設計

## 1. 基本資料

- 頁面：`A02.jsp`
- 對應 BDD / SDD / TDD：`a02-personal-report`
- 使用者：一般員工、學顧、行政

## 2. 頁面目的

- 查詢自己的日報與週報，並由入口頁進入新增、修改或檢視

## 3. 版面區塊

- 查詢列：起日、迄日、填報人姓名、搜尋按鈕
- 日報表格區：`A02_table_daily`，表格列由 Ajax 查詢結果動態 append，顯示日報日期、事由與操作鈕
- 日報新增按鈕列：學顧日報 / 行政日報
- 週報表格區：`A02_table_weekly`，表格列由 Ajax 查詢結果動態 append，顯示週報週期、事由與操作鈕
- 週報新增按鈕列：學顧週報 / 行政週報

## 4. 欄位與顯示規則

- 首次載入會帶入預設日期區間並自動查詢
- 日報與週報固定分成兩張表，不混在同一列表
- 行政與學顧資料以不同配色顯示類型
- 草稿狀態顯示 `修改 Edit`，送審後顯示 `檢視 View`
- 日報與週報的列內容、操作按鈕與顯示順序都由 JavaScript 依回傳資料重建

## 5. 按鈕與動作

- `搜尋`：查詢日報與週報
- 四個新增按鈕：分別帶入報表類型與角色
- `修改 / 檢視`：以報表編號進入明細頁

## 6. 互動規則

- 日期未完整輸入不得查詢
- 首次進入會呼叫 `AjaxSearchDefault`
- 搜尋後以 Ajax 同步重建日報與週報兩張表
- 每筆結果會依 `re_type` 分流到日報或週報表格，再動態組出 `修改 / 檢視` 按鈕

## 7. 訊息設計

- 日期未輸入完整
- 查無資料
- Ajax 查詢失敗

## 8. 權限與狀態

- 僅查詢與操作本人資料
- 依報表狀態決定是否允許修改
