# A03 請假申請 SDD

## 1. 對應需求

- Story：`A03-1`、`A03-2`
- 對應 BDD：`bdd/a03-leave-request.feature`
- 目標：讓員工建立、查詢、修改、送審與取消自己的請假單

## 2. 角色

- 申請人：建立草稿、送審、查詢歷史、取消單據
- 系統：產生單號、保存狀態、控制可編輯條件

## 3. 畫面與路由

- `A03.jsp`：請假入口、歷史列表、查詢條件、建立入口
- `A030.jsp`：請假編輯/檢視頁
- 常見互動：`AjaxSearchDefault`、`AjaxSearch`、`AjaxDelete`

## 4. 資料設計

- 主檔：`leave` 或對應請假申請資料表
- 核心欄位：單號、申請人、請假日期區間、假別、事由、附件、狀態碼、送審時間

## 5. 流程設計

1. 使用者進入 `A03.jsp` 載入最近單據與預設查詢結果
2. 使用者進入 `A030.jsp` 填寫假別、日期、事由後先儲存草稿
3. 按下送審後狀態由 Draft 轉為 Submitted
4. 草稿或可取消單據可由本人刪除或取消

## 6. 權限與前端互動

- 僅可查看與操作本人單據
- 送審前需完成日期、假別與事由驗證
- 列表保留狀態、單號、建立時間與明細入口
- 已送審或已結案資料不可再任意編修

## 7. 錯誤處理

- 日期區間不合法不得送出
- 必填欄位缺漏不得送審
- 非本人或狀態不允許時不得刪除/取消

## 8. 可測試點

- 草稿建立與修改
- 送審狀態轉換
- 本人資料隔離
- 刪除/取消限制
