# B01 電訪紀錄 SDD

## 1. 對應需求

- Story：`B01-1`
- 對應 BDD：`bdd/b01-interview-record.feature`
- 目標：追蹤電訪歷程、查詢結果與後續跟進狀態

## 2. 角色

- 業務人員：建立與更新電訪紀錄
- 主管：查詢團隊進度、檢視結果分布
- 系統：保存聯繫結果、下次追蹤時間與備註

## 3. 畫面與路由

- `B01.jsp`：電訪紀錄列表與查詢入口
- `B010.jsp` / `B011.jsp`：明細與編修頁

## 4. 資料設計

- 主檔：`interview_record`
- 關聯：`lead`
- 核心欄位：名單、聯絡結果、追蹤日期、下次聯絡時間、備註、負責人

## 5. 流程設計

1. 使用者由列表查詢待追蹤名單
2. 進入明細頁新增或更新電訪結果
3. 系統依結果回寫學員狀態與下次追蹤節點

## 6. 權限與前端互動

- 業務僅可維護自己負責資料
- 主管可查看本區或團隊資料
- 列表需顯示追蹤結果、下次聯絡時間與明細入口

## 7. 錯誤處理

- 聯絡結果與追蹤時間缺漏不得送出
- 無權限者不得編修他人資料

## 8. 可測試點

- 查詢條件生效
- 電訪結果保存
- 下次追蹤資料回寫
- 角色資料範圍
