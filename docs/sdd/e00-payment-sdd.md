# E00 繳費查詢與雙重確認 SDD

## 1. 對應需求

- Story：`E00-1`、`E00-2`
- 關聯前端：`E00.jsp`
- 目標：完成繳費資料查詢、建立、財務確認、教務確認與退回

## 2. 角色

- 財務人員：查詢、建立繳費、財務確認、退回
- 教務人員：教務確認、必要時退回
- 業務或其他角色：僅查看授權範圍資料，不得執行確認

## 3. 畫面與路由

### 畫面

- `E00.jsp`：搜尋區、結果表格、確認/退回按鈕

### 已知路由

- `GET /E/E00`
- `POST /E/E00`
- `POST /E/E00/AjaxSearch`
- `POST /E/E00/Finance`
- `POST /E/E00/Academic`
- `POST /E/E00/Reject`

## 4. 資料設計

### 主要資料表

- `payment_info`：繳費主檔
- 關聯學員、課程、機構資料表：供查詢條件與顯示用

### 核心欄位

- 學員 ID / 姓名
- 區域、機構、課程、負責學顧
- 金額
- 幣別
- 繳費日期
- `pay_fin`
- `pay_aca`
- 退回原因

## 5. 狀態設計

本流程主要不是單一狀態碼，而是兩個確認欄位組合：

| pay_fin | pay_aca | 說明 |
|---|---|---|
| 0 | 0 | 新建未確認 |
| 1 | 0 | 財務已確認 |
| 0 | 1 | 教務已確認 |
| 1 | 1 | 雙方皆完成，視為完成 |
| 任意 | 任意 + reject reason | 已退回，需重處理 |

## 6. 流程設計

### 查詢流程

1. 進入 `E00.jsp`
2. 選擇學生姓名、區域、機構、課程、學顧等條件
3. 呼叫 `AjaxSearch`
4. 顯示結果表與確認狀態

### 建立流程

1. 財務人員選定學員
2. 輸入金額、幣別、日期
3. 送出 `POST /E/E00`
4. 寫入 `payment_info`

### 確認流程

1. 財務人員送出 `Finance`
2. 教務人員送出 `Academic`
3. 兩方都完成後視為結案

### 退回流程

1. 任一授權角色送出 `Reject`
2. 系統保存退回原因
3. 畫面標示為需重新處理

## 7. 權限規則

- 財務角色可建立與財務確認
- 教務角色可執行教務確認
- 退回動作限授權角色
- 已完成資料不得被未授權角色重寫

## 8. 前端互動設計

- 搜尋條件應支援多欄位篩選
- session 記錄最後搜尋條件
- 確認/退回按鈕依角色與資料狀態顯示
- 更新後畫面需回傳最新確認欄位狀態

## 9. 錯誤處理

- 建立資料時欄位缺漏不得儲存
- 重複確認需被阻擋
- 退回時應要求輸入原因
- 未授權角色呼叫確認 API 應直接拒絕

## 10. 可測試點

- 新增 payment_info
- 財務確認
