# F00 區域管理 SDD

## 1. 對應需求

- Story：`F00-1`
- 對應 BDD：`bdd/f00-area-management.feature`
- 目標：維護區域主檔與啟用狀態，供組織結構與權限資料範圍使用

## 2. 角色

- 系統管理員：新增、修改、停用區域
- 一般員工或主管：不得維護區域
- 系統：驗證區域代碼唯一性與欄位完整性

## 3. 畫面與路由

- `F00.jsp`：區域列表
- `F000.jsp`：區域編修頁

## 4. API 對應

- `GET /regions`：查詢區域清單
- `POST /regions`：建立區域
- `PUT /regions/{regionId}`：修改區域

## 5. 資料設計

- 主檔：`region`
- 核心欄位：區域代碼、區域名稱、狀態、建立人、更新時間
- 區域代碼需全系統唯一

## 6. 流程設計

1. 管理員進入區域清單查詢現有資料
2. 管理員於編修區新增或修改區域資料
3. 系統驗證必填欄位與區域代碼唯一性
4. 驗證通過後儲存，並回傳最新清單

## 7. 權限與前端互動

- 僅系統管理角色可執行新增、編修、儲存
- 列表需顯示區域狀態與編修入口
- 非授權角色僅可瀏覽或直接拒絕進入

## 8. 錯誤處理

- 區域代碼重複不得儲存
- 必填欄位缺漏不得儲存
- 無權限者不得修改

## 9. 風險與設計注意事項

- 區域停用後，關聯資料範圍設定需避免選入停用區域
- 區域代碼若被外部主檔引用，需維持唯一且不可重複建立

## 10. 可測試點

- 區域新增成功
- 區域修改成功
- 唯一性驗證
- 管理權限限制
