# C04 學員管理 SDD

## 1. 對應需求

- Story：`C04-1`
- 對應 BDD：`bdd/c04-student-management.feature`
- 目標：維護學員基本資料、課程狀態、繳費與學顧分配

## 2. 角色

- 教務人員：查詢與修改學員資料
- 學顧/業務：更新服務相關欄位
- 系統：動態帶出課程、繳費項目與 popup 操作

## 3. 畫面與路由

- `C04.jsp`：學員列表與查詢入口
- `C041.jsp` ~ `C046.jsp`：基本資料、繳費、課程、學顧分配與明細

## 4. 資料設計

- 主檔：`student`
- 關聯：`student_course`、`payment_info`、`advisor_assign`
- 核心欄位：學員姓名、聯絡方式、分校、課程、學顧、繳費狀態、課程狀態

## 5. 流程設計

1. 使用者於 `C04.jsp` 以多條件查詢學員
2. 進入子頁維護基本資料、繳費或課程狀態
3. 變更課程時前端同步重建繳費項目
4. 必要時以 popup 變更學顧或查看明細

## 6. 權限與前端互動

- 區域/機構/課程採連動下拉
- 課程勾選會動態刷新付款項目
- 金額、日期、匯率等欄位先做前端驗證
- 不同角色顯示不同子頁與按鈕

## 7. 錯誤處理

- 資料格式錯誤不得儲存
- 無權限者不得修改不屬於自己範圍的學員
- 未選課程不得建立繳費資料

## 8. 可測試點

- 學員查詢
- 子頁切換與資料保存
- 動態繳費項目
- 學顧調整流程
