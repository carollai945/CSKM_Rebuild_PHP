# A00 個人資料維護 SDD

## 1. 對應需求

- Story：`A00-1`、`A00-2`
- 對應 BDD：`bdd/a00-personal-data.feature`
- 目標：讓員工維護個人資料、照片、家庭資料與工作經歷

## 2. 角色

- 員工：編修自己的個資與照片
- 系統：載入既有資料、執行欄位驗證並保存更新結果
- 權限管理流程：可唯讀開啟此頁供檢視

## 3. 畫面與路由

- `A00.jsp`：個人資料維護頁
- `POST /A/A00`：保存個人資料與照片
- `GET /api/v1/me/personal-data`：載入本人個資與照片
- `PUT /api/v1/me/personal-data`：更新本人個資
- `POST /api/v1/me/personal-data/photo`：更新本人照片
- `GET /api/v1/staff/{staffId}/personal-data?mode=readonly`：由 F02 帶入唯讀檢視
- 關聯資源：員工守則 PDF、照片顯示頁

## 4. 資料設計

- 主檔：`hr_management` 或對應人員資料表
- 關聯：照片、教育程度、家庭資料、工作經歷
- 核心欄位：姓名、聯絡方式、身高體重、血型、性別、照片、學歷、技能、家庭與工作經歷

## 5. 流程設計

1. 員工進入頁面時載入既有個資與照片
2. 員工可更新個人資料並選擇照片檔案
3. 前端先做數字與欄位長度驗證
4. 驗證通過後呼叫 `PUT /api/v1/me/personal-data`，成功則提示並重新整理頁面
5. 若有新照片則呼叫 `POST /api/v1/me/personal-data/photo`
6. 由 F02 帶入時以 `mode=readonly` 載入，狀態固定為唯讀

## 6. 權限與前端互動

- 血型與性別下拉於載入時自動帶入目前值
- 日期欄位使用日期選擇器
- 同上按鈕可將戶籍地址帶入通訊地址
- 閱讀員工守則按鈕另開 PDF
- 唯讀模式需隱藏導覽列與操作按鈕

## 7. 錯誤處理

- 手機、體重等欄位非數字時不得儲存
- 各欄位超過長度限制時應停在第一個錯誤欄位
- 儲存失敗時應顯示失敗訊息

## 8. 可測試點

- 個資載入
- 同上按鈕
- 唯讀模式
- 驗證與儲存成功/失敗
- API 權限：本人可寫、管理者代查唯讀
