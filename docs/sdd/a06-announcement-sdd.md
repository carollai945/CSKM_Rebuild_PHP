# A06 公告管理 SDD

## 1. 對應需求

- Story：`A06-1`、`A06-2`
- 對應 BDD：`bdd/a06-announcement.feature`
- 目標：讓使用者建立公告、引用簽呈、送審與查詢公告歷程

## 2. 角色

- 公告建立者：新增公告、引用已核准簽呈、送出公告
- 主管/CEO：審核公告
- 系統：控制引用模式、欄位唯讀狀態與公告狀態

## 3. 畫面與路由

- `A06.jsp`：公告入口、歷史列表、查詢條件
- `A060.jsp`：公告編輯頁
- `A061.jsp`：公告檢視頁
- 關聯審核頁：`D02.jsp`、`D020.jsp`

## 4. 資料設計

- 主檔：`announcement`
- 關聯：`petition`
- 核心欄位：公告編號、主旨、內容、引用簽呈編號、狀態、發布範圍、附件

## 5. 流程設計

1. 使用者進入 `A06.jsp` 查詢或建立公告
2. 一般建立模式可直接輸入主旨與內容
3. 引用簽呈模式僅能選擇本人已核准簽呈，並鎖定帶入內容
4. 送審後進入公告審核流程

## 6. 權限與前端互動

- 引用簽呈後主旨/內容轉為唯讀
- 畫面需隱藏一般送審按鈕，改以引用公告流程送出
- 送公告前需再次確認
- 列表顯示狀態、建立人與明細入口

## 7. 錯誤處理

- 不可引用他人或未核准簽呈
- 主旨或內容缺漏不得送出
- 狀態不允許時不得修改已送審資料

## 8. 可測試點

- 一般公告建立
- 引用簽呈鎖定欄位
- 送審與退回重送
- 公告檢視流程
