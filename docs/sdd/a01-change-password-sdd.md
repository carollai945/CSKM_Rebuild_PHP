# A01 修改密碼 SDD

## 1. 對應需求

- Story：`A01-1`
- 對應 BDD：`bdd/a01-change-password.feature`
- 目標：讓員工以原密碼驗證後修改自己的登入密碼

## 2. 角色

- 員工：輸入舊密碼與新密碼完成修改
- 系統：驗證舊密碼、比對新密碼一致性並保存新密碼

## 3. 畫面與路由

- `A01.jsp`：修改密碼頁
- `POST /A/A01`：提交密碼變更

## 4. 資料設計

- 主檔：登入帳號與密碼資料
- 核心欄位：用戶帳號、舊密碼、新密碼、驗證密碼

## 5. 流程設計

1. 使用者進入頁面確認帳號
2. 輸入舊密碼與兩次新密碼後送出
3. 系統驗證舊密碼與新密碼一致性
4. 成功則更新密碼，失敗則返回錯誤訊息

## 6. 權限與前端互動

- 頁面固定顯示目前登入帳號
- 錯誤訊息顯示於表單下方

## 7. 錯誤處理

- 舊密碼錯誤不得更新
- 兩次新密碼不一致不得更新

## 8. 可測試點

- 正常修改
- 舊密碼錯誤
- 驗證密碼不一致
