# B02 學員服務管理 SDD

## 1. 對應需求

- Story：`B02-1`
- 對應 BDD：`bdd/b02-student-service.feature`
- 目標：讓業務/學顧維護學員服務、課程與繳費相關紀錄

## 2. 角色

- 業務/學顧：維護學員服務紀錄、建立繳費項目、更新課程狀態
- 主管：查看轄區資料與服務品質
- 系統：依課程動態重建繳費項目並驗證輸入

## 3. 畫面與路由

- `B02.jsp`：學員查詢與入口
- `B020.jsp` ~ `B023.jsp`：服務紀錄、繳費、課程狀態等子頁

## 4. 資料設計

- 主檔：`student_service`
- 關聯：`student`、`payment_info`、`course_status`
- 核心欄位：學員、負責人、服務備註、課程、繳費項目、金額、狀態

## 5. 流程設計

1. 使用者查詢學員並進入服務子頁
2. 勾選課程後前端重建可選繳費項目
3. 使用者建立服務備註、繳費資料或課程狀態
4. 資料保存後回到學員服務列表

## 6. 權限與前端互動

- 課程變更時即時重建付款項目
- 提供項目全選/全不選
- 備註、項目、金額需先通過前端驗證
- 角色不同時僅顯示可操作子頁與按鈕

## 7. 錯誤處理

- 未選課程或未選繳費項目不得送出
- 金額不合法不得儲存
- 無權限者不得修改他區或他人負責學員

## 8. 可測試點

- 動態項目重建
- 服務紀錄保存
- 繳費資料建立
- 角色差異與資料範圍
