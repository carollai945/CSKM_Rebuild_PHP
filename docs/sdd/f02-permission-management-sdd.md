# F02 人員帳號與權限管理 SDD

## 1. 對應需求

- Story：`F02-2`、`F02-3`、關聯 `F02-4`
- 關聯前端：`F02.jsp`、`F020.jsp`
- 目標：讓系統管理員可建立員工帳號、管理頁面權限、重設密碼與停用帳號

## 2. 角色

- 系統管理員：新增員工、修改權限、刪除/停用、重設密碼
- 一般員工：不得存取 F02
- 系統：依組織結構提供部門與職稱 Ajax 選單

## 3. 畫面與路由

### 畫面

- `F02.jsp`：既有人員查詢與權限編輯主畫面
- `F020.jsp`：新增人員畫面

### 已知路由

- `GET /F/F02`
- `POST /F/F02`
- `GET /F/F020`
- `POST /F/F020`
- `POST /F/F020/save`
- `POST /F/F02/search`
- `POST /F/F02/gettitle`
- `POST /F/F02/getdep`
- `POST /F/F02/delete`
- `POST /F/F02/reset`

## 4. 資料設計

### 主要資料表

- `hr_management`：員工主檔
- `menu` 或同類授權資料：頁面權限
- `branch_mgmt`：區域資料
- `department_mgmt`：部門與職稱資料

### 核心欄位

- 員工編號、姓名、英文名、性別、血型
- 區域、部門、職稱
- 特殊權限：`priv_ceo`、`priv_regmgr`、`priv_finmgr`、`priv_staff`
- 離職日期 `l_date`
- 初始密碼 / 重設密碼

## 5. 流程設計

### 新增人員

1. 管理員進入 `F020.jsp`
2. 選擇區域後 Ajax 取得部門列表
3. 選擇部門後 Ajax 取得職稱列表
4. 輸入基本資料、角色權限、頁面權限
5. 送出 `F020/save`
6. 建立 `hr_management` 與權限資料

### 修改權限

1. 管理員在 `F02.jsp` 搜尋員工
2. 系統帶出人員資料與既有勾選權限
3. 管理員修改角色旗標與頁面勾選
4. 送出 `POST /F/F02`
5. 系統更新人員與權限資料

### 刪除 / 停用 / 重設密碼

- `delete`：刪除或標記停用
- `reset`：重設密碼
- `l_date`：離職後登入應被拒絕

## 6. 權限規則

- 僅系統管理員可使用 F02 / F020
- 一般員工不得修改他人頁面權限
- 從 `F05` 連入 `F02` 時，仍須再次檢查 F02 權限

## 7. 前端互動設計

- 角色權限與頁面權限同頁管理
- `getdep`：依區域動態載入部門
- `gettitle`：依部門動態載入職稱
- 搜尋後需保留已勾選權限狀態
- 權限更新應讓下次登入立即生效

## 8. 錯誤處理

- 重複員工編號不得建立
- 區域/部門/職稱不完整不得儲存
- 無權限者應直接拒絕
- 刪除或停用中的帳號不應可再次被正常登入

## 9. 風險與設計注意事項

- F02 是全系統權限中心，錯誤設定會影響所有模組
- 角色旗標與頁面勾選若不同步，容易出現 UI 有選單但無法操作的問題
- Ajax 選單資料若失真，會導致新增人員資料不完整
