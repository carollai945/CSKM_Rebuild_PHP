# D00 請假審核 SDD

## 1. 對應需求

- Story：`D00-1`
- 對應 BDD：`bdd/d00-leave-approval.feature`
- 目標：讓主管與 CEO 在待審清單中完成單筆或批次請假審核

## 2. 角色

- 區域主管：審核主管階段待審請假單
- CEO：審核最終待核資料
- 系統：依角色切換按鈕、維護批次回覆結果與錯誤清單

## 3. 畫面與路由

- `D00.jsp`：待審清單、查詢區、批次回覆區
- `D000.jsp`：單筆審核明細頁
- 互動：`AjaxSearch`、`AjaxSelect`、`saveMultiReply`

## 4. 資料設計

- 主檔：`leave`
- 核心欄位：單號、申請人、請假區間、目前狀態、主管意見、CEO 意見、審核時間

## 5. 流程設計

1. 主管或 CEO 進入 `D00.jsp` 查詢待審請假單
2. 可選擇進入 `D000.jsp` 逐筆審核
3. 也可於列表勾選多筆資料、輸入共同意見後批次回覆
4. 系統回傳成功結果或失敗單號清單

## 6. 權限與前端互動

- 主管只看主管階段待審；CEO 只看已經主管核准的資料
- 清單提供全選/全不選
- 主管與 CEO 勾選集合彼此獨立，不可混用
- 批次回覆需依角色帶入不同 Type 值

## 7. 錯誤處理

- 未勾選資料不得批次送出
- 退回時未填意見不得送出
- 已被他人處理或狀態改變的單據應回報失敗單號

## 8. 可測試點

- 單筆核准/退回
- 批次回覆
- 全選/全不選
- 失敗清單回饋
