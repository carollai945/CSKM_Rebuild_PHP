# D02 公告審核 SDD

## 1. 對應需求

- Story：`D02-1`
- 對應 BDD：`bdd/d02-announcement-approval.feature`
- 目標：讓主管與 CEO 依角色完成公告審核與退回

## 2. 角色

- 區域主管：審核主管階段公告
- CEO：最終核准公告
- 系統：依狀態與角色顯示可操作按鈕

## 3. 畫面與路由

- `D02.jsp`：公告待審清單
- `D020.jsp`：公告審核明細

## 4. 資料設計

- 主檔：`announcement`
- 核心欄位：公告編號、主旨、內容、建立人、狀態、主管意見、CEO 意見

## 5. 流程設計

1. 審核者於列表查詢待審公告
2. 進入明細查看公告內容與引用來源
3. 送出核准或退回後回寫狀態與意見

## 6. 權限與前端互動

- 主管與 CEO 僅能看到自己階段資料
- 依角色顯示不同核准/退回按鈕
- 審核頁應保留返回清單與意見欄位

## 7. 錯誤處理

- 退回未填意見不得送出
- 狀態不符不得審核
- 無權限者不得進入審核頁

## 8. 可測試點

- 主管核准/退回
- CEO 核准/退回
- 權限與狀態限制
