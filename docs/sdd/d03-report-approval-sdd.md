# D03 報表審核 SDD

## 1. 對應需求

- Story：`D03-1`
- 對應 BDD：`bdd/d03-report-approval.feature`
- 目標：審核日報、週報等工作報表並區分角色與報表種類

## 2. 角色

- 主管：審核第一層報表
- CEO：審核最終報表
- 系統：依報表別與狀態顯示對應明細頁

## 3. 畫面與路由

- `D03.jsp`：報表待審列表
- `D030.jsp` ~ `D033.jsp`：不同報表明細與審核頁

## 4. 資料設計

- 主檔：`daily_report`、`weekly_report` 或對應報表表
- 核心欄位：報表編號、報表類型、填報人、期間、內容摘要、狀態、審核意見

## 5. 流程設計

1. 審核者先於清單辨識報表類型
2. 系統導向對應的明細頁
3. 審核者填入意見並核准或退回

## 6. 權限與前端互動

- 列表需清楚顯示報表類型與期間
- 不同報表子頁保留不同欄位內容，但審核動作一致
- 角色僅處理自己階段資料

## 7. 錯誤處理

- 報表類型不符時不得導向錯誤頁面
- 退回未填意見不得送出
- 已審核報表不得重複審核

## 8. 可測試點

- 報表別導頁
- 主管/CEO 分層審核
- 已審核鎖定
