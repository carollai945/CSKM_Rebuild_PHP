# A05 請款申請 SDD

## 1. 對應需求

- Story：`A05-1`
- 對應 BDD：`bdd/a05-invoice-request.feature`
- 目標：讓員工建立請款資料並送入財務確認流程

## 2. 角色

- 申請人：建立請款單、補附件、送審、查詢歷史
- 系統：保存請款內容、串接財務確認、回寫狀態

## 3. 畫面與路由

- `A05.jsp`：請款入口、歷史清單、建立入口
- `A050.jsp`：請款編輯/檢視頁
- 後續關聯頁：`E02.jsp`、`E03.jsp`

## 4. 資料設計

- 主檔：`invoice_request` 或對應請款資料表
- 核心欄位：請款單號、申請人、請款項目、金額、說明、附件、狀態、退回原因

## 5. 流程設計

1. 使用者自 `A05.jsp` 建立請款草稿
2. 於 `A050.jsp` 填寫明細、附件與金額後送審
3. 單據進入財務列表與確認流程
4. 若被退回，申請人依退回原因修正後重送

## 6. 權限與前端互動

- 僅申請人可編修自己的草稿或退回單
- 列表應顯示目前狀態與退回原因摘要
- 送審前需驗證金額、請款事由與必要附件

## 7. 錯誤處理

- 金額格式錯誤不得儲存
- 缺少必要欄位或附件不得送審
- 已進入財務處理的單據不得由申請人刪除

## 8. 可測試點

- 草稿建立
- 送審與退回重送
- 附件保存
- 財務流程狀態銜接
